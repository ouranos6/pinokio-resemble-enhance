# Repository Standards & Conventions

## 1. Scope and Purpose

- This file captures repo-specific conventions that keep the Pinokio launcher reproducible and prevent drift between the launcher repo and the upstream app clone.

## 2. Core Conventions

### Canonical Upstream Patches in `patches/`

**Status:** REQUIRED

**Scope:** Pinokio scripts ([install.json](install.json), [update.json](update.json)) and any modifications to the upstream `resemble-enhance` code.

**Rule:**

- Treat `patches/` as the single source of truth for upstream code overrides; `install.json` and `update.json` must re-apply these files into `app/` on every install/update.

**Rationale (Why this exists):**

- `app/` is (re)generated by cloning/pulling upstream, so direct edits inside `app/` can be overwritten.
- The launcher needs deterministic behavior across fresh installs (Batch UI + DeepSpeed-safe inference).

**Examples:**

- Good:
  - Update a file under `patches/resemble_enhance/...` and let `install.json`/`update.json` copy it into `app/...`.
- Bad:
  - Editing `app/resemble-enhance/...` only (changes can disappear after `update.json`).

**Related Files / Modules:**

- [install.json](install.json)
- [update.json](update.json)
- patches/app.py
- patches/resemble_enhance/**

## 3. Rationale and Examples

- The upstream repo layout can be either `app/app.py` (flat clone) or `app/resemble-enhance/app.py` (nested). Scripts should handle both.

## 4. Known Exceptions

- None currently.
